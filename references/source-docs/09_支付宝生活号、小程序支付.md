# 支付宝生活号、小程序支付

> 通过本接口获取支付宝交易凭证，在支付宝生活号（公众号）或小程序中调起支付宝收银台完成支付。

---

## 1 接口说明

**请求地址**：`POST http://api2.lfwin.com/payapi/trade/alipay`

**适用场景**：
- **生活号支付**：在支付宝生活号（类似公众号）中调起支付
- **小程序支付**：在支付宝小程序中调起支付

**service 取值**：

| 场景 | service 值 |
|------|-----------|
| 生活号支付 | `comm.js.pay` |
| 小程序支付 | `comm.mini.pay` |

---

## 2 请求参数

| 参数名 | 类型 | 必需 | 描述 |
|--------|------|------|------|
| service | string | 是 | 接口名称（`comm.js.pay` 或 `comm.mini.pay`） |
| apikey | string | 是 | 商户 APIKEY，唯一标识，由服务商提供 |
| money | string | 是 | 支付金额，单位：元，最小 0.01 |
| nonce_str | string | 是 | 随机字符串，长度不超过32位 |
| sign | string | 是 | 签名，参照签名算法 |
| buyer_id | string | 是 | 支付宝用户 buyer_id（即 userid，2088 开头） |
| buyer_open_id | string | 否 | 小程序内获取的用户 openid，与 buyer_id 二选一（暂时仅支持官方通道） |
| sub_appid | string | 否 | 商户的小程序 APPID |
| mch_orderid | string | 否 | 商户外部订单号，必须保证每次下单唯一；下单失败时请调用查询接口 |
| remarks | string | 否 | 订单备注 |
| notify_url | string | 否 | 异步通知地址（完整URL，带参数需整个网址 urlencode 编码） |
| order_url | string | 否 | 商户小程序订单详情页跳转地址 |
| goods_info | string | 否 | 商品详情，单品优惠活动该字段必传；JSON字符串格式 |
| split_info | string | 否 | 分账规则（JSON格式） |
| split | string | 否 | 分账标识，`1`：标识订单分账 |
| attach | string | 否 | 附加数据，支付成功回调时原样返回，长度 1-128 字符，`=`、`&`、`?` 字符请编码 |

---

## 3 请求示例

```
service=comm.mini.pay&apikey=00014005&money=0.01&nonce_str=123456&sign=b0b12b65e6347c8d47923e7b2d53d7cc&buyer_id=2088102754687385&mch_orderid=20180209000000001&notify_url=https://api.xxx.com/notify
```

---

## 4 返回参数

| 参数名 | 描述 | 类型 |
|--------|------|------|
| service | 接口名称 | string |
| version | 接口版本号：3.0 | number |
| charset | 字符编码：UTF-8 | string |
| message | 错误信息 | string |
| status | 错误状态 | number |
| sign_type | 签名方式 | string |
| sign | 签名字符串 | string |
| trade_no | 通道流水号，用于调起支付宝收银台 | string |
| orderid | 系统订单号，唯一值 | string |
| order_time | 下单时间（时间戳，秒） | string |
| mch_orderid | 商户外部订单号 | string |

## 6 唤起支付方式

### 生活号支付

参照支付宝官方文档：[唤起收银台 JSAPI](https://opendocs.alipay.com/open/common/105591)

### 小程序支付

参照支付宝官方接口：[alipay.trade.create（统一收单交易创建接口）](https://opendocs.alipay.com/apis/api_1/alipay.trade.create)

---

## 7 goods_info 商品详情格式

> 单品优惠活动时该字段**必传**，JSON字符串格式

```json
[
  {"goods_id": "apple-01", "goods_name": "iPhone6s 32G", "quantity": "1", "price": "528800"},
  {"goods_id": "apple-02", "goods_name": "iPhone6s 64G", "quantity": "1", "price": "608800"}
]
```

| 字段 | 说明 |
|------|------|
| goods_id | 商品编码（大小写字母、数字、中划线、下划线，string） |
| goods_name | 商品实际名称（string） |
| quantity | 购买数量（string） |
| price | 商品单价，**单位：分**（string）；有优惠时传优惠后的单价 |

---

## 8 split_info 动态分账格式

> JSON字符串格式，分账规则如下

```json
[
  {"mer_email": "youlian313@163.com", "money": "0.01"},
  {"mer_email": "haoyunlai@126.com", "money": "0.02"}
]
```

| 字段 | 说明 |
|------|------|
| mer_email | 渠道商户邮箱 |
| money | 分账金额，单位：元 |

---

## 9 注意事项

1. `buyer_id` 与 `buyer_open_id` **二选一**即可（`buyer_id` 为支付宝 userid，2088开头；`buyer_open_id` 仅支持官方通道小程序）
2. 下单后如果 **15 分钟** 未支付，系统会自动关闭订单
3. 支付结果以**异步通知**为准，商户需提供可访问的 `notify_url`
4. `attach` 字段中 `=`、`&`、`?` 字符请做好 URL 编码
