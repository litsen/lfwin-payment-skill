## 1.5 签名算法

### 通用规则

- `sign` 字段本身**不参与**签名计算。
- 除 `sign` 外，所有提交的参数（含空值参数）均参与签名。
- 字段按 **ASCII 码从小到大**排序，用 `key=value&key=value…` 格式拼接。
- 拼接完成后，**末尾追加** `&signkey=商户密钥`。
- 签名原始串中字段名和字段值均**不进行 URL Encode**。
- 平台可能随时增加返回参数，验签时禁止写死参数列表。

---

### MD5 签名步骤

| 步骤 | 操作 |
|------|------|
| **Step 1** | 取出所有请求参数（不含 `sign`），有值即参与 |
| **Step 2** | 按字段名 ASCII 升序排列，用 `key=value&` 格式拼接 |
| **Step 3** | 末尾拼接 `&signkey=商户密钥` |
| **Step 4** | 对拼接好的字符串执行 **MD5 加密**（32位小写） |
| **Step 5** | 将 MD5 结果作为 `sign` 参数值提交（`signkey` 不提交） |

**示例（扫码支付）**：

请求参数：
```
service=pay.alipay.qrcode&apikey=00014005&money=0.01&nonce_str=123456
```

签名原始串：
```
apikey=00014005&money=0.01&nonce_str=123456&service=pay.alipay.qrcode&signkey=punr8ucu
```

MD5 结果：
```
sign = e2bc07b313c49f166ec03796370d8eb2
```

---

### MD5 验签步骤

| 步骤 | 操作 |
|------|------|
| **Step 1** | 取出响应中所有参数（不含 `sign`），含新增参数 |
| **Step 2** | 按字段名 ASCII 升序排列，用 `key=value&` 格式拼接 |
| **Step 3** | 末尾拼接 `&signkey=商户密钥` |
| **Step 4** | 对拼接好的字符串执行 **MD5 加密**（32位小写） |
| **Step 5** | 将计算结果与响应中 `sign` 值比对，相等则验签通过 |

> ⚠️ 若 `status ≠ 10000`，通常表示请求失败，**无需验签**。

---

### RSA 签名步骤

| 步骤 | 操作 |
|------|------|
| **Step 1** | 取出所有请求参数（不含 `sign`） |
| **Step 2** | 按字段名 ASCII 升序排列，用 `key=value&` 格式拼接 |
| **Step 3** | 使用**商户 RSA 私钥**对排序后的字符串执行 RSA 签名 |
| **Step 4** | 将签名结果作为 `sign` 参数值提交 |

### RSA 验签步骤

| 步骤 | 操作 |
|------|------|
| **Step 1** | 取出响应中所有参数（不含 `sign`） |
| **Step 2** | 按字段名 ASCII 升序排列，用 `key=value&` 格式拼接 |
| **Step 3** | 使用**平台 RSA 公钥**验证 `sign` 签名 |
| **Step 4** | 验签结果为 `true` 则通过 |

> ⚠️ **警告**：验签时**一定不要用商户 RSA 公钥**，必须使用平台提供的服务端 RSA 公钥。

---

### 签名注意事项

1. `sign` 字段**不参与**签名；除 `sign` 外所有参数均参与（含空值参数）。
2. 字段按 **ASCII 升序**排列，拼接格式为 `key1=value1&key2=value2`。
3. 拼接完成后**末尾追加** `&signkey=商户密钥`（MD5）；RSA 无需拼接 signkey。
4. 签名原始串中**不进行 URL Encode**，使用原始值。
5. 平台可能随时**增加返回参数**，验签时禁止写死参数列表。
6. `notify_url` 参数需在签名**之前**做 `urldecode` 解码。
7. 只要参与提交的参数都要参与签名；例如 `a=&b=1`，则 `a=` 也需进入待签名字符串。
8. 数组类型参数（如 `fund_bill_list`）验签时需先将其 **JSON 序列化为字符串**后参与签名。
9. `status ≠ 10000` 的响应通常**不签名**，无需验签。

## 1.6 全局状态码

| 状态码 | 描述 |
|--------|------|
| `10000` | SUCCESS（成功） |
| `1002` | 暂无记录 |
| `1020` | 接口暂不支持 |
| `1900` | 请在微信或支付宝中打开扫码 |
| `2900` | 请求执行错误 |
| `3000` | apikey 错误 |
| `3010` | 请求参数错误 |
| `3020` | 支付授权码格式错误 |
| `3030` | 订单号格式错误 |
| `3040` | 订单号不存在 |
| `3050` | 请求过于频繁 |
| `3060` | 时间参数不支持跨月 |
| `3900` | **验签失败** |
| `4000` | 接口返回错误 |
| `4001` | 退款处理中 |
| `4002` | 商户余额不足，退款失败 |
| `4800` | 订单更新失败，请重新更新订单 |
| `4900` | 订单错误 |
| `4910` | 订单已撤销或已退款 |
| `4920` | 订单待支付，无法调用退款 |
| `4930` | 订单已支付，无法调用撤销 |
| `4940` | 订单非该设备 apikey 收款，无法调用退款或撤销 |
| `4950` | 订单未支付成功，无法调用预授权撤销或完成 |
| `4960` | 订单已关闭 |
| `4970` | 分账订单无法调用退款或撤销 |
| `4980` | 分账接收方不存在 |
| `4990` | 退款订单号有误 |
| `5010` | 商家状态不正常 |
| `5020` | 代理状态不正常 |
| `5030` | 设备商户号不存在或与配置不一致 |
| `5060` | 店员状态不正常 |
| `6000` | 账号或密码不正确 |
| `9999` | 系统错误 |
