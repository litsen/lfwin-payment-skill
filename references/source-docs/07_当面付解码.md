# 当面付解码

> 在发起条码支付之前，先通过本接口将顾客出示的付款码（`dynamic_id`）解析为用户身份标识（`user_id` / `openid`），再用于后续支付扣款。

---

## 1 接口说明

**请求地址**：`POST /payapi/pay/decode_bar`  
**service**：`pay.alipay.decode_bar`

---

## 2 请求参数

| 参数名 | 类型 | 必需 | 描述 |
|--------|------|------|------|
| service | string | 是 | 接口名称，固定值：`pay.alipay.decode_bar` |
| apikey | string | 是 | 商户 APIKEY，唯一标识 |
| dynamic_id | string | 是 | 顾客付款码（条码设备扫描获取） |
| nonce_str | string | 是 | 随机字符串（≤32位） |
| sign | string | 是 | 签名，参照签名算法 |
| sence_no | string | 否 | 解码外部业务号，同一付款码重复解码时必须与上一次保持一致；后续调用条码支付时必须传相同值 |
| sub_appid | string | 否 | 微信子商户公众账号ID（微信渠道时使用） |

### 请求示例

```
service=pay.alipay.decode_bar&apikey=00014005&dynamic_id=286103991909766417&nonce_str=123456&sign=2171ed542106e5a26eb0ae7d1f9a7a22
```

---

## 3 返回参数

| 参数名 | 描述 | 类型 |
|--------|------|------|
| service | 接口名称 | string |
| sence_no | 解码外部业务号 | number |
| user_id | 支付宝用户 userId 或微信 openid | string |
| openid | 微信用户在 appid 下的唯一标识（微信渠道） | string |
| sub_openid | 微信用户在 sub_appid 下的唯一标识 | string |
| sub_appid | 微信分配的子商户公众账号ID | string |
| appid | 微信分配的商户号 | string |
| version | 接口版本号：3.0 | number |
| charset | 字符编码：UTF-8 | string |
| message | 错误信息 | string |
| status | 错误状态 | number |
| sign_type | 签名方式 | string |
| sign | 签名字符串 | string |

### 返回示例

```json
{
    "service": "pay.alipay.decode_bar",
    "user_id": "2088102176697083",
    "openid": "",
    "sub_openid": "",
    "sub_appid": "",
    "appid": "",
    "status": "10000",
    "message": "SUCCESS"
}
```

---

## 4 详细说明

### 支付宝当面付解码

1. 支付宝当面付付款码解码功能需要商户**签约该产品**后方可使用。
2. 通过授权码查询后，该授权码只能由此商户号发起扣款，**直至授权码更新**。

### 微信解码规则

| 子商户号绑定情况 | sub_appid 传参 | 返回字段 |
|----------------|---------------|--------|
| 未绑定 appid | 非必传 | 仅返回 `openid` |
| 绑定**一个** appid | 非必传 | 返回 `openid` + `sub_openid` |
| 绑定**多个** appid | **必传** | 返回 `openid` + `sub_openid` |

---

## 5 使用流程

```
1. 收银设备扫描顾客付款码（dynamic_id）
       ↓
2. 调用本接口解码，获取 user_id / openid
       ↓
3. 使用获取到的 user_id/openid 调用条码支付接口完成扣款
```
