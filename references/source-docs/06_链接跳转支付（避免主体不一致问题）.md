# 链接跳转支付（避免主体不一致问题）

> 生成支付跳转链接，适用于公众号内唤起支付（H5支付）的场景。

---

## 1 接口说明

> 生成跳转链接后，将链接展示给用户或在 WebView 中打开，用户完成支付后平台回调通知商户。

**请求地址**：`POST /payapi/pay/jspay3`

### 支持通道一览

| 通道类型 | service 值 |
|----------|-----------|
| 支付宝直连 | `alipay.comm.jspay` |
| 微信直连 | `wxpay.comm.jspay` |
| 云闪付 | `unpay.comm.jspay` |

---

## 2 请求参数

| 参数名 | 类型 | 必需 | 描述 |
|--------|------|------|------|
| service | string | 是 | 接口名称（见上表） |
| apikey | string | 是 | 商户 APIKEY |
| money | string | 是 | 支付金额，单位：元，最小 0.01 |
| nonce_str | string | 是 | 随机字符串（≤32位） |
| sign | string | 是 | 签名 |
| mch_orderid | string | 否 | 商户外部订单号（唯一） |
| notify_url | string | 否 | 异步通知地址（带参数需 urlencode） |
| remarks | string | 否 | 订单备注 |
| store_dis | string | 否 | 平台店铺优惠活动开关：`1` 打开，`0` 关闭（默认） |
| time_expire | string | 否 | 订单失效时间（目前仅支持微信官方通道，时间戳秒；刷卡≥1分钟，其他≥5分钟，最长2小时） |
| sign_type | string | 否 | 签名方式：MD5/RSA，默认 MD5 |
| split_info | string | 否 | 动态分账规则（JSON格式） |

### 请求示例

```
service=wxpay.comm.jspay&apikey=00014005&money=0.01&nonce_str=123456&sign=b0b12b65e6347c8d47923e7b2d53d7cc&mch_orderid=20180209000000001&notify_url=http://www.xxx.com
```

---

## 3 返回参数

| 参数名 | 描述 | 类型 |
|--------|------|------|
| url | H5 支付跳转页面地址 | string |
| orderid | 系统订单号（唯一） | string |
| mch_orderid | 商户外部订单号 | string |
| service | 接口名称 | string |
| money | 支付金额（单位：元） | number |
| order_time | 下单时间（时间戳，秒） | string |
| version | 接口版本号：3.0 | number |
| charset | 字符编码：UTF-8 | string |
| message | 错误信息 | string |
| status | 错误状态 | number |
| sign_type | 签名方式 | string |
| sign | 签名字符串 | string |

### 返回示例

```json
{
    "url": "http://api2.lfwin.com/index/jspay/pay2?orderid=202004151400341658962365",
    "orderid": "202004151400341658962365",
    "mch_orderid": "",
    "service": "wxpay.comm.jspay",
    "money": "0.01",
    "order_time": "1586930434",
    "status": "10000",
    "message": "SUCCESS"
}
```

---

## 4 注意事项

- 需在后台提前配置平台的APPID和支付目录
- 公众支付安全目录 → 配置为 `http://www.lfwin.com/wxpay/`
