# 异步通知

> 所有支付接口在生产环境下，若传了 `notify_url` 参数，支付/退款成功时平台会自动回调通知商户系统。

> ⚠️ **重要**：不保证通知最终一定能成功，在订单状态不明或者没有收到通知的情况下，建议商户主动调用**支付状态轮询接口**确认订单状态，不要过于依赖回调。

---

## 2.1 支付成功异步通知

> 请求方式：POST，参数以 QueryString 表单作为请求 body 发送到 `notify_url`

### 通知参数

| 参数名 | 类型 | 必需 | 描述 |
|--------|------|------|------|
| orderid | string | 是 | 系统订单号，唯一值 |
| trade_no | string | 是 | 通道流水号 |
| dis_name | string | 是 | 通道方订单号 |
| paystatus | string | 是 | 支付状态：`1` 支付成功，`0` 待付款，`2` 付款失败 |
| paymoney | string | 是 | 下单支付金额，单位：元 |
| pri_paymoney | string | 是 | 商家优惠活动后实际支付金额，单位：元 |
| order_time | string | 是 | 下单时间，时间戳（秒） |
| paytime | string | 是 | 支付成功时间，时间戳（秒） |
| mch_orderid | string | 是 | 商户外部订单号 |
| notify_url | string | 是 | 通知地址 |
| buyer_account | string | 是 | 买家账号（微信/支付宝账号） |
| attach | string | 否 | 附加数据，下单时若传此参数，通知将原样返回 |
| sign | string | 是 | 签名字符串 |

### 通知返回格式

```
orderid=20180202171044007002473&trade_no=4200000061201802025681296197&dis_name=201802021723331617316118&paystatus=1&paymoney=0.01&pri_paymoney=0.01&order_time=1517562644&paytime=1517562682&mch_orderid=&buyer_account=微信账号&notify_url=http%3A%2F%2Fwww.xxx.com&sign=0ee2e3d34573de75f973630403db24b6
```

### 商户回调要求

> 商户收到通知后，**必须输出 `success`**（7个字符，不含引号），否则系统认为通知失败，会继续重试。

重试策略：约 4 次，时间点为第 1、5、15、30 分钟。

## 2.2 退款完成异步通知

> 请求方式：POST，参数以 QueryString 表单发送到 `notify_url`

### 通知参数

| 参数名 | 类型 | 必需 | 描述 |
|--------|------|------|------|
| refund_orderid | string | 是 | 系统退款订单号，唯一值 |
| orderid | string | 是 | 原支付订单系统订单号 |
| addtime | number | 是 | 退款下单时间，时间戳（秒） |
| notify_url | string | 是 | 通知地址 |
| rt_trade_no | string | 是 | 通道退款单号 |
| mch_refund_no | string | 是 | 商户退款单号 |
| refundmoney | string | 是 | 退款金额，单位：元 |
| refund_handmoney | string | 是 | 退款手续费，单位：元 |
| refund_status | string | 是 | 退款状态：`1` 成功，`0` 处理中，`2` 失败 |
| refundtime | string | 是 | 退款完成时间，时间戳（秒） |
| attach | string | 否 | 附加数据，退款时若传此参数，通知将原样返回 |
| sign | string | 是 | 签名字符串 |

---

## 2.3 异步通知验签

### 判断验签类型

根据返回的 `sign_type` 参数判断：
- `sign_type == RSA` → 使用 **RSA 验签**
- 其他（默认）→ 使用 **MD5 验签**

### MD5 验签步骤

| 步骤 | 操作 |
|------|------|
| **Step 1** | 取出通知中除 `sign` 外的所有参数（含新增参数，**不要写死参数列表**） |
| **Step 2** | 按字段名 ASCII 升序排列，用 `key=value&` 拼接 |
| **Step 3** | 末尾拼接 `&signkey=商户密钥` |
| **Step 4** | 对字符串执行 **MD5 加密**（32位小写） |
| **Step 5** | 计算结果与通知中 `sign` 比对，一致则验签通过 |

**示例**：

通知原始串：
```
orderid=201909061609431364590937&trade_no=4200000401201909061532983835&paystatus=1&paymoney=0.10&pri_paymoney=0.10&order_time=1567757383&paytime=1567757386&mch_orderid=7777M51909060005&notify_url=http%3a%2f%2fwww.baidu.com&buyer_account=微信&sign=99d2f41d1df83670b80dd9d7e8bc0939
```

去除 sign，排序后拼接 signkey：
```
buyer_account=微信&mch_orderid=7777M51909060005&notify_url=http://www.baidu.com&order_time=1567757383&orderid=201909061609431364590937&paymoney=0.10&paystatus=1&paytime=1567757386&pri_paymoney=0.10&trade_no=4200000401201909061532983835&signkey=punr8ucu
```

MD5 计算结果 = `99d2f41d1df83670b80dd9d7e8bc0939` → **验签通过** ✅

### RSA 验签步骤

| 步骤 | 操作 |
|------|------|
| **Step 1** | 取出通知中除 `sign` 外的所有参数 |
| **Step 2** | 按字段名 ASCII 升序排列，用 `key=value&` 拼接 |
| **Step 3** | 使用**平台 RSA 公钥**验签 |
| **Step 4** | 验签结果为 `true` 则通过 |

> ⚠️ **一定不要用商户 RSA 公钥**，必须使用平台提供的服务端 RSA 公钥。
