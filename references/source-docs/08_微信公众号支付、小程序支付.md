# 微信公众号支付、小程序支付

> 通过 JSAPI 接口获取微信支付所需的完整参数，在微信内置浏览器（公众号）或小程序中调起支付收银台。

---

## 1 接口说明

**请求地址**：`POST http://api2.lfwin.com/payapi/mini/wxpay`

**适用场景**：
- **公众号支付**（JSAPI）：在微信内置浏览器中打开网页完成支付
- **小程序支付**（Mini Program）：在小程序中调起微信支付

**service 取值**：

| 场景 | service 值 | 说明 |
|------|-----------|------|
| 公众号支付 | `comm.js.pay` | 在微信内置浏览器中调起 |
| 小程序支付 | `comm.mini.pay` | 在小程序中调起 |

> 均需提前在后台配置好对应的 **APPID**，公众号支付需配置**支付授权目录**。

---

## 2 请求参数

| 参数名 | 类型 | 必需 | 描述 |
|--------|------|------|------|
| service | string | 是 | 接口名称（`comm.js.pay` 或 `comm.mini.pay`） |
| apikey | string | 是 | 商户 APIKEY，唯一标识，由服务商提供 |
| money | string | 是 | 支付金额，单位：元，最小 0.01 |
| nonce_str | string | 是 | 随机字符串，长度不超过32位 |
| sub_appid | string | 是 | 小程序/公众号的 appid（需与 sub_openid 对应，且 appid 需与微信支付商户号关联） |
| sub_openid | string | 是 | 消费者在 sub_appid 下的唯一标识 |
| sign | string | 是 | 签名，参照签名算法 |
| mch_orderid | string | 否 | 商户外部订单号，必须保证每次下单唯一；下单失败时请调用查询接口 |
| notify_url | string | 否 | 异步通知地址（完整URL，带参数需整个网址 urlencode 编码） |
| time_expire | string | 否 | 订单失效时间（时间戳，单位秒；微信官方/合利宝通道：默认15分钟，官方最长2小时，合利宝最长24小时；刷卡至少1分钟） |
| goods_info | string | 否 | 商品详情，单品优惠活动该字段必传；JSON字符串格式 |
| goods_tag | string | 否 | 微信官方订单优惠标记（代金券或立减）；设置后需支付金额满足条件才触发代金券 |
| sub_wx_mchid | string | 否 | 指定微信子商户号交易（合利宝/乐刷资金管控渠道使用） |
| sign_type | string | 否 | 签名方式：MD5/RSA，默认 MD5 |
| split_info | string | 否 | 动态分账规则（JSON格式） |
| split | string | 否 | 分账标识，`1`：标识订单分账（乐刷） |
| attach | string | 否 | 附加数据，支付成功回调时原样返回，长度 1-128 字符，`=`、`&`、`?` 字符请编码 |
| good_name | string | 否 | 订单标题，不可使用特殊字符（如 `/`、`=`、`&` 等） |

---

## 3 请求示例

```
service=comm.mini.pay&apikey=00014005&money=0.01&nonce_str=123456&sub_appid=wxxxxxxx&sub_openid=oxxxxxxx&sign=b0b12b65e6347c8d47923e7b2d53d7cc&mch_orderid=20180209000000001&notify_url=http://www.xxx.com/notify
```

---

## 4 返回参数

| 参数名 | 描述 | 类型 |
|--------|------|------|
| service | 接口名称 | string |
| version | 接口版本号：3.0 | number |
| charset | 字符编码：UTF-8 | string |
| message | 错误信息 | string |
| status | 错误状态 | number |
| sign_type | 本接口签名方式（不要与 signType 混淆） | string |
| sign | 签名字符串 | string |
| appId | 调起支付的微信公众号/小程序 APPID | string |
| timeStamp | 调起支付的时间戳（从1970年1月1日起的秒数） | string |
| nonceStr | 调起支付的随机字符串 | string |
| signType | 调起支付的签名算法类型 | string |
| package | 调起支付参数，格式为 `prepay_id=*` | string |
| paySign | 调起支付的签名 | string |
| orderid | 系统订单号，唯一值 | string |
| order_time | 下单时间（时间戳，秒） | number |
| mch_orderid | 商户外部订单号 | string |
| qr_code | 二维码链接 | string |
| trade_type | 交易类型（如 `JSAPI`） | string |
| prepay_id | 预支付交易会话标识 | string |

## 6 唤起支付方式

### 公众号支付（H5）

```javascript
// 示例（具体请参照微信官方文档）
WeixinJSBridge.invoke(
  'getBrandWCMPayRequest', {
    "appId": res.appId,       // 公众号 appId
    "timeStamp": res.timeStamp, // 时间戳
    "nonceStr": res.nonceStr,   // 随机字符串
    "package": res.package,      // prepay_id
    "signType": res.signType,    // 签名方式
    "paySign": res.paySign       // 支付签名
  },
  function(res) {
    // 支付成功/失败回调
  }
);
```

> ⚠️ **注意点**：
> 1. 前端 `timeStamp` 参数**区分大小写**（Vue 中建议改为 `timestamp`）
> 2. 返回值中有 `signType`（调起支付参数）和 `sign_type`（本接口自身签名方式），**切勿混淆**

> 公众号支付授权目录需在后台 → 通道管理中配置APPID和支付目录

### 小程序支付

参照微信官方 SDK 文档：[小程序调起支付](https://pay.weixin.qq.com/wiki/doc/api/wxa/wxa_api.php?chapter=7_7&index=5)

## 7 goods_info 商品详情格式

> 单品优惠活动时该字段**必传**，JSON字符串格式

```json
[
  {"goods_id": "apple-01", "goods_name": "iPhone6s 32G", "quantity": "1", "price": "528800"},
  {"goods_id": "apple-02", "goods_name": "iPhone6s 64G", "quantity": "1", "price": "608800"}
]
```

| 字段 | 说明 |
|------|------|
| goods_id | 商品编码（大小写字母、数字、中划线、下划线，string） |
| goods_name | 商品实际名称（string） |
| quantity | 购买数量（string） |
| price | 商品单价，**单位：分**（string）；有优惠时传优惠后的单价 |

---

## 8 split_info 动态分账格式

> JSON字符串格式，分账规则如下

```json
[
  {"account": "E1800305780", "amount": "10.00"},
  {"account": "E1321546", "amount": "0.02"}
]
```

| 字段 | 说明 |
|------|------|
| account | 分账商户号（不可传发起方自身） |
| amount | 分账金额，单位：元 |
| is_original | 可选；传 `1` 表示接收方收到的金额为扣掉手续费后的值，不传则按原始分账金额分给接收方 |

> ⚠️ 一笔订单可以传多个接收方，接收方不可重复

---

## 9 订单超时时间

系统默认15分钟关闭订单

| 通道 | 最长超时时间 |
|------|---------|
| 微信官方 | 默认15分钟，最长2小时 |
| 合利宝 | 默认15分钟，最长24小时 |
| 其他 | 默认15分钟 |

> 例：`time_expire=1565618931` 表示订单在 `2019-08-12 22:08:51` 过期

---

## 10 常见报错

| 报错 | 原因 | 处理方式 |
|------|------|---------|
| `5035, sub mch id 与 sub appid 不匹配` | 通道配置问题 | 联系客服配置支付通道的 appid 和支付目录（能返回此错误说明接口已调通） |
| `调用支付JSAPI缺少参数：total_fee` | 前端未正确赋值支付参数 | 检查前端是否正确将参数赋值到支付页面 |
| `当前页面的URL未注册` | 支付目录未配置 | 联系客服配置 appid 和支付目录 |
