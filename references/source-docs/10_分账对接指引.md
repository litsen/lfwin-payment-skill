# 分账功能对接

> 用于将订单金额按规则分账给多个接收方。
> 支持自动分账（下单时传入 split_info）和延迟分账（支付后通过接口发起分账）两种模式。

---

## 1 分账对接指引

### 1.1 分账前置流程

1. 联系业务开通分账功能
2. 参与分账的接收方需进件获取相应商户号
3. 接收方如果只作为接收使用、不含收款功能，可使用**接收方通用材料进件**：
   - 身份证和银行卡
   - 三合照传小微分账接收方专用图替代（门头照、收银台照、内景照）
4. 接收方均需进行**分账接收方绑定**（后台或调用接口添加）

![接收方通用材料进件](https://img.10tu.net/lfwin/images/live/share_guide1.png)

### 1.2 订单自动分账场景

适用于门店固定比例分给固定接收方的情况。可直接使用后台的**门店自动分账功能**配置完成。

### 1.3 订单动态分账场景

适用于接收方和金额都不固定的情况，需使用接口对接，分两种方式：

**方式一：支付完成后自动分账**

- 通过支付订单创建类接口下单时，增加 `split_info` 参数标记需要分账的对象和金额
- 订单支付成功后系统自动执行分账

**方式二：延迟分账**

- 下单时增加 `split=1` 参数标记订单为分账订单
- 待订单完成支付后，调用方通过【分账请求】接口传入自定义分账明细 `split_info`，即时执行分账
- 可通过【分账查询】接口查询分账结果

> ⚠️ **下单请求不可同时传 split 和 split_info**

### 1.4 split_info 分账规则格式

```json
[
  {"account": "E1800305780", "amount": "10.00"},
  {"account": "E1321546", "amount": "0.02"}
]
```

| 字段 | 说明 |
|------|------|
| account | 分账商户号（**不可传发起方自身**） |
| amount | 分账金额，单位：元 |
| is_original | 可选；传 `1` 表示接收方收到的金额为扣掉手续费后的值，不传则按原始分账金额分给接收方 |

> ⚠️ 一笔订单可以传多个接收方，接收方不可重复

### 1.5 订单分账退款流程

1. 调用【退款请求】接口，传入 `split_info` 字段，完成分账退回和订单退款
2. 支持部分退款，第三方需自行计算好分账退款出资方，且要保证出资方余额充足
3. 通过【退款查询】接口查询/更新退款订单执行状态

## 2 分账方关联

> 添加分账接收方商户，建立分账发起方与接收方的关联关系。

**请求地址**：`POST http://api2uat.lfwin.com/payapi/pay/sharing_receiver_bind`  
**service**：支持多个通道alipay|lspay|lspay2|heli|yop，传参如：`pay.lspay.sharing_receiver_bind`

### 请求参数

| 参数名 | 类型 | 必需 | 描述 |
|--------|------|------|------|
| service | string | 是 | 接口名称：`pay.lspay.sharing_receiver_bind` |
| apikey | string | 是 | 商户 APIKEY，唯一标识，由服务商提供 |
| nonce_str | string | 是 | 随机字符串，长度不超过32位 |
| sign | string | 是 | 签名，参照签名算法 |
| sharing_merchant_id | string | 是 | 分账发起方商户号 |
| receiver_merchant_id | string | 是 | 分账接收方商户号 |
| receiver_merchant_name | string | 是 | 分账接收方名称 |
| receiver_merchant_type | string | 否 | 分账接收方类型（乐刷可置空） |
| protocol_pic | string | 否 | 协议资源列表（多协议以英文逗号分隔，上传后获取通道资源ID；仅乐刷通道需要） |
| remark | string | 否 | 说明 |

### 返回参数

| 参数名 | 描述 | 类型 |
|--------|------|------|
| version | 接口版本号：3.0 | number |
| charset | 字符编码：UTF-8 | string |
| message | 错误信息 | string |
| status | 错误状态 | number |
| sign_type | 签名方式：MD5 | string |
| sign | 签名字符串 | string |

### 返回示例

```json
{
    "version": "3.0",
    "charset": "UTF-8",
    "message": "SUCCESS",
    "status": "10000",
    "sign_type": "MD5",
    "sign": "3ac0a8f3803bf511f053534d7b254ed1"
}
```

## 5 分账请求

> 对已支付订单发起分账，执行分账金额分配。

**请求地址**：`POST http://api2uat.lfwin.com/payapi/pay/order_sharing`  
**service**：支持多个通道alipay|lspay|lspay2|heli|yop，传参如：`pay.lspay.order_sharing`

### 请求参数

| 参数名 | 类型 | 必需 | 描述 |
|--------|------|------|------|
| service | string | 是 | 接口名称：`pay.lspay.order_sharing` |
| apikey | string | 是 | 商户 APIKEY，唯一标识，由服务商提供 |
| orderid | string | 是 | 系统订单号，唯一值 |
| nonce_str | string | 是 | 随机字符串，长度不超过32位 |
| sign | string | 是 | 签名，参照签名算法 |
| money | string | 是 | 订单金额，单位：元 |
| mch_orderid | string | 否 | 商户订单号（与 orderid 同时存在时，以 orderid 为准） |
| order_time | string | 否 | 下单时间，时间戳，精确到秒（只传商户订单号时必传） |
| version | string | 否 | 版本号，不传默认 3.0 |
| sign_type | string | 否 | 签名方式：MD5/RSA，默认 MD5 |
| split_info | string | 是 | 自定义分账执行详情，json格式（见下方说明） |

### split_info 格式

**分账1人：**
```json
[{"account":"5215519353","amount":"1.25","remark":"测试"}]
```

**分账2人：**
```json
[
  {"account":"5215519353","amount":"1.25","remark":"测试1"},
  {"account":"5215519351","amount":"1.95","remark":"测试2"}
]
```

> 合利宝和乐刷通道支持 `is_original` 可选参数

### 返回参数

| 参数名 | 描述 | 类型 |
|--------|------|------|
| service | 接口名称 | string |
| orderid | 系统订单号，唯一值 | string |
| sharing_orderid | 乐刷分账订单号 | string |
| out_order_no | 系统分账订单号 | string |
| version | 接口版本号：3.0 | string |
| charset | 字符编码：UTF-8 | string |
| message | 错误信息 | string |
| status | 错误状态（10000 表示提交成功，待查询分账结果） | number |
| sign_type | 签名方式：MD5 | string |
| sign | 签名字符串 | string |
| mch_orderid | 商户订单号 | string |

### 返回示例

```json
{
    "service": "pay.lspay.order_sharing",
    "orderid": "202304041341251205180930",
    "sharing_orderid": "000000200005323094",
    "out_order_no": "sh202304041341251205180930",
    "mch_orderid": "",
    "version": "3.0",
    "charset": "UTF-8",
    "message": "SUCCESS",
    "status": "10000",
    "sign_type": "MD5",
    "sign": "8e0bc6c680aa03f83e6529b8fd1d35d7"
}
```

### 注意事项

- 目前支持官方支付宝、微信、乐刷（乐刷联合收单）、合利宝、易宝通道
- 不同通道资金进入余额可能有延迟，建议在支付成功 **30秒后** 再发起分账请求
- 如需支付成功后立即分账，建议走订单分账流程

## 7 图片资料上传通道

> 目前仅乐刷通道需要上传协议图片

> 上传分账协议图片，获取通道资源ID，用于分账方关联接口的 `protocol_pic` 字段。

**请求地址**：`POST http://api2uat.lfwin.com/merchant/register/pic_nosql`  
**service**：`lspay.apply.pic`

> ⚠️ **签名说明**：该接口与支付业务不同，不需要传 `apikey`，而要传 `agent_num`，签名密钥请联系技术获取。

### 请求参数

| 参数名 | 类型 | 必需 | 描述 |
|--------|------|------|------|
| service | string | 是 | 接口名称：`heli.apply.pic` |
| agent_num | string | 是 | 代理商编号 |
| nonce_str | string | 是 | 随机字符串，长度不超过32位 |
| sign | string | 是 | 签名，参照签名算法 |
| req_no | string | 是 | 上送渠道系统流水号（已进件商户上送渠道成功后获取，请联系技术获取） |
| pic | string | 是 | 上传图片文件（PHP用 `new CURLFile(pic图片路径)` 上传） |
| mer_id | string | 否 | 商户进件渠道后返回的渠道商户号 |
| pic_http_url | string | 否 | 上传图片文件在线地址（与 pic 二选一，同时存在优先 pic） |

### pic 参数上传注意事项

PHP 上传示例：
```php
new CURLFile(pic图片路径)
```

### 返回参数
