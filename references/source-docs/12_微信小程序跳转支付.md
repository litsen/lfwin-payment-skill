# 微信小程序跳转支付

> 适用于 H5 或 APP 场景：商户服务端调用 LFWin 接口获取微信小程序 Scheme 或 URL Link，前端控制浏览器访问返回链接，从而唤起微信小程序并完成支付。

---

## 1 接口说明

**请求地址**：`POST http://api2.lfwin.com/payapi/mini/mini_url`

**service 取值**：`comm.mini.url`

**适用场景**：

- H5 页面、APP WebView 或 APP 外部浏览器中跳转微信小程序支付。
- 通过微信小程序 Scheme 或 URL Link 唤起指定小程序。
- 交通类商户需要在小程序内通过 `wx.navigateToMiniProgram` 跳转支付。

> 本接口返回的 `url` 是用于唤起小程序的链接。顾客取消支付后如需重新支付，必须重新调用本接口创建新订单，不要重复访问旧的 `weixin://` 链接，否则容易出现“商户订单号已存在”。

---

## 3 请求示例

```text
service=comm.mini.url&apikey=00014005&money=0.01&nonce_str=123456&url_link=1&sign=示例签名
```

## 4 返回参数

| 参数名 | 类型 | 描述 |
|---|---|---|
| url | string | 微信小程序链接；访问后将唤起小程序。可能是 `weixin://...`、`https://wxaurl.cn/...` 或 `https://wxmpurl.cn/...` |
| orderid | string | LFWin 系统订单号，唯一值 |
| money | string | 支付金额，单位元 |
| mch_orderid | string | 商户外部订单号，必须保证每次下单唯一 |
| service | string | 接口名称 |
| mini_appid | string | 用于小程序内跳转的 appid |
| mini_pagePath | string | 用于小程序内跳转的页面路径 |
| mini_orderNo | string | 用于小程序内跳转的订单号 |
| version | string | 接口版本号，通常为 `3.0` |
| charset | string | 字符编码，通常为 `UTF-8` |
| message | string | 返回信息 |
| status | string | 状态码，`10000` 表示请求成功 |
| sign_type | string | 本接口签名方式 |
| sign | string | 签名字符串，按通用签名方法验签 |

## 6 url_link 参数

| 取值 | 返回类型 | 说明 |
|---|---|---|
| 不传或 `0` | 小程序 Scheme | 适用于短信、邮件、外部网页、微信内等拉起小程序场景。生成结果通常形如 `weixin://dl/business/?t=*TICKET*` |
| `1` | 小程序 URL Link | 适用于短信、邮件、网页、微信内等拉起小程序场景。生成结果通常形如 `https://wxaurl.cn/*TICKET*` 或 `https://wxmpurl.cn/*TICKET*` |

小程序 Scheme 和 URL Link 目前主要面向国内非个人主体小程序开放，支持生成到期失效或永久有效的小程序链接能力，具体限制以微信官方能力为准。

---

## 7 小程序 Scheme 与 URL Link 区别

### Scheme

生成格式示例：

```text
weixin://dl/business/?t=*TICKET*
```

- iOS 系统支持识别 URL Scheme，可在短信等应用场景中直接通过 Scheme 跳转小程序。
- Android 系统不支持直接识别 Scheme。若要在 Android 上唤起小程序，需要制作 H5 页面并在页面中执行跳转。

跳转示例：

```html
<script>
location.href = 'weixin://dl/business/?t=*TICKET*'
</script>
```

也可以改为用户点击按钮后再触发跳转。

### URL Link

生成格式示例：

```text
https://wxaurl.cn/*TICKET*
https://wxmpurl.cn/*TICKET*
```

URL Link 更适合放在短信、邮件、网页和微信内链接中打开。

## 8 H5、APP 支付流程

微信小程序跳转支付流程：

1. 商户后端调用本接口创建支付订单并获取 `url`。
2. H5 或 APP 控制浏览器访问返回的 `url`。
3. 系统唤起微信小程序。
4. 用户在小程序内完成支付。
5. 商户后端通过异步通知和主动查单确认支付结果。

支付宝 H5/APP 场景可使用扫码支付（顾客扫商家动态二维码）能力：APP 控制浏览器跳转扫码支付接口返回的 `qr_code` 参数，唤起支付宝支付。

## 9 自有服务商渠道代理商对接建议

如果是自有服务商渠道代理商对接 H5/APP 微信支付，可按以下方式自行生成小程序跳转链接：

1. 申请一个与服务商主体或商户主体一致的小程序。服务商主体小程序可适用于服务商渠道下所有商户；商户主体小程序只能适用于该商户。
2. 对接微信官方“获取 URL Scheme”或“获取 URL Link”接口，二选一即可。
3. 先调用微信“获取调用凭据”接口获取 `access_token`。
4. 生成跳转链接时，在 `jump_wxa` 结构中传入小程序 `path` 和 `query`。`path` 是要跳转的小程序路径，`query` 是跳转携带参数。
5. 可通过 `query` 携带 `apikey` 等业务参数给小程序接收，再由小程序内服务端逻辑请求支付接口。
6. 调用成功后得到形如 `weixin://dl/business/?t=tvyFIomSZjg` 的链接，APP 或 H5 控制浏览器跳转该链接唤起小程序。
7. 小程序接收到 `path` 和 `query` 参数后，再对接“微信公众号、小程序支付”接口，在小程序内完成支付。

微信官方文档：

- 获取 URL Scheme：`https://developers.weixin.qq.com/miniprogram/dev/api-backend/open-api/url-scheme/urlscheme.generate.html`
- 获取 URL Link：`https://developers.weixin.qq.com/miniprogram/dev/api-backend/open-api/url-link/urllink.generate.html`
- 获取调用凭据：`https://developers.weixin.qq.com/miniprogram/dev/api-backend/open-api/access-token/auth.getAccessToken.html`

## 10 小程序内跳转支付

交通类商户通过本接口也支持小程序内跳转支付。返回参数带 `mini_appid`、`mini_pagePath`、`mini_orderNo` 时，可在小程序内通过如下方式唤起小程序支付：

```javascript
wx.navigateToMiniProgram({
  appId: mini_appid,
  path: mini_pagePath,
  extraData: {
    orderNo: mini_orderNo
  },
  success(res) {
    // 打开成功
  }
})
```

## 11 联调检查清单

- 使用通用签名和验签规则，`signkey` 只参与签名，不随请求提交。
- `url_link=0` 或不传时返回 Scheme，`url_link=1` 时返回 URL Link。
- 顾客取消支付后重新支付时，重新调用本接口创建新订单。
- 不要把本接口的跳转支付和 `/payapi/mini/wxpay` 小程序内直接调起支付混用。
- 创建订单成功后保存 `orderid`、`mch_orderid`、`url`、`mini_appid`、`mini_pagePath`、`mini_orderNo`。
- 支付结果以异步通知和 `pay.comm.query_order` 主动查单为准，不以前端跳转结果作为最终成功依据。
