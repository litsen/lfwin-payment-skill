# 扫码支付（C扫B）

> 商户生成二维码展示给顾客，顾客使用微信/支付宝/云闪付扫一扫完成支付。

---

## 1 聚合扫码支付

> 生成不指定支付方式的二维码，顾客扫码后系统自动判断并调起对应支付。商户只需生成一个二维码，支持微信、支付宝、云闪付三种方式。

**请求地址**：`POST /payapi/trans/kxpay`  
**service**：`pay.comm.jspay`

### 请求参数

| 参数名 | 类型 | 必需 | 描述 |
|--------|------|------|------|
| service | string | 是 | 接口名称，固定值：`pay.comm.jspay` |
| apikey | string | 是 | 商户 APIKEY |
| money | string | 是 | 支付金额，单位：元，最小 0.01 |
| nonce_str | string | 是 | 随机字符串（≤32位） |
| sign | string | 是 | 签名 |
| mch_orderid | string | 否 | 商户外部订单号（唯一） |
| notify_url | string | 否 | 异步通知地址（完整URL，带参数需 urlencode） |
| succ_url | string | 否 | 支付成功跳转页面地址 |
| err_url | string | 否 | 支付取消/失败返回链接 |
| remarks | string | 否 | 订单备注 |
| store_dis | string | 否 | 平台店铺优惠活动开关：`1` 打开，`0` 关闭 |
| sign_type | string | 否 | 签名方式：MD5/RSA，默认 MD5 |
| goods_info | string | 否 | 商品详情（JSON格式，单品优惠必传） |
| split_info | string | 否 | 动态分账规则（JSON格式） |
| split | string | 否 | 分账标识（乐刷） |
| attach | string | 否 | 附加数据（回调时原样返回，1-128字符） |

### 请求示例

```
service=pay.comm.jspay&apikey=00014005&money=0.01&nonce_str=123456&sign=b0b12b65e6347c8d47923e7b2d53d7cc&mch_orderid=20180209000000001
```

### 返回参数

| 参数名 | 描述 | 类型 |
|--------|------|------|
| url | 聚合支付跳转地址（生成二维码） | string |
| orderid | 系统订单号（唯一） | string |
| mch_orderid | 商户外部订单号 | string |
| order_time | 下单时间（时间戳，秒） | string |
| service | 接口名称 | string |
| version | 接口版本号：3.0 | number |
| charset | 字符编码：UTF-8 | string |
| message | 错误信息 | string |
| status | 错误状态 | number |
| sign_type | 签名方式 | string |
| sign | 签名字符串 | string |

### 返回示例

```json
{
    "service": "pay.comm.jspay",
    "orderid": "201908131050120070023965",
    "mch_orderid": "",
    "order_time": "1565664612",
    "url": "http://api2.lfwin.com/index/jspay/kx/orderid/201908131050120070023965",
    "status": "10000",
    "message": "SUCCESS"
}
```

### 注意事项

- 将 `url` 生成二维码展示给顾客即可
- 用户扫码前 `m_paytype` 为未知（如 `comm`）；扫码后变为实际支付方式（如 `wxpay`）
- 商户收银软件需**轮询订单状态**（`pay.comm.query_order`）判断用户是否已完成支付
- 二维码默认 **15分钟** 超时，超时后不可支付
- 测试环境**不发送异步通知**

## 2 指定通道扫码支付/支付宝链接跳转支付

> 若需明确指定某一通道，可使用对应的 service 值，覆盖场景与聚合扫码支付完全一致。

| 支付方式 | service 值 | 请求路径 |
|----------|-----------|----------|
| 支付宝 | `pay.alipay.qrcode` | `/payapi/pay/qrcode` |
| 微信支付 | `pay.wxpay.qrcode` | `/payapi/pay/qrcode` |
| 云闪付 | `pay.unpay.qrcode` | `/payapi/pay/qrcode` |

> 以下参数为各通道扫码支付的**完整请求参数**（各通道通用），如字段无需使用可不传。

### 请求参数

| 参数名 | 类型 | 必需 | 描述 |
|--------|------|------|------|
| service | string | 是 | 接口名称 |
| apikey | string | 是 | 商户 APIKEY |
| money | string | 是 | 支付金额，单位：元，最小 0.01 |
| nonce_str | string | 是 | 随机字符串（≤32位） |
| sign | string | 是 | 签名 |
| mch_orderid | string | 否 | 商户订单号（唯一） |
| notify_url | string | 否 | 异步通知地址（完整URL，带参数需 urlencode） |
| fq_num | string | 否 | 分期期数 |
| seller_percent | string | 否 | 商家承担分期手续费比例（%） |
| mid | string | 否 | 商户 MID |
| goods_tag | string | 否 | 商品标签 |
| remarks | string | 否 | 订单备注 |
| guid | string | 否 | 全局唯一标识 |
| store_dis | string | 否 | 平台店铺优惠活动开关：`1` 打开，`0` 关闭 |
| limitpay | string | 否 | 限制支付 |
| alipay_store_id | string | 否 | 支付宝门店ID |
| time_expire | string | 否 | 订单失效时间（微信官方/合利宝通道，时间戳秒，默认15分钟） |
| goods_info | string | 否 | 商品详情（JSON格式，单品优惠必传） |
| term_info | string | 否 | 终端信息 |
| split_info | string | 否 | 动态分账规则（JSON格式） |
| sign_type | string | 否 | 签名方式：MD5/RSA，默认 MD5 |
| attach | string | 否 | 附加数据（回调时原样返回） |
| share_model | string | 否 | 分账模式 |
| fee_ratio_list | string | 否 | 手续费比例列表 |
| share_merchant_id | string | 否 | 分账商户ID |
| score_money | string | 否 | 积分抵扣金额 |
| good_name | string | 否 | 商品名称 |

### 返回参数

| 参数名 | 描述 | 类型 |
|--------|------|------|
| service | 接口名称 | string |
| orderid | 系统订单号 | string |
| qr_code | 支付二维码内容（二维码原始链接） | string |
| paymoney | 下单支付金额（单位：元） | number |
| pri_paymoney | 实际支付金额（单位：元） | number |
| paystatus | 支付状态：`1` 成功，`0` 待付款，`2` 失败 | number |
| order_time | 下单时间（时间戳，秒） | number |
| mch_orderid | 商户外部订单号 | string |
| code_url | 二维码在线地址（可放入 img 标签展示） | string |
| version | 接口版本号：3.0 | number |
| charset | 字符编码：UTF-8 | string |
| message | 错误信息 | string |
| status | 错误状态 | number |
| sign_type | 签名方式 | string |
| sign | 签名字符串 | string |

### 返回示例

```json
{
    "service": "pay.alipay.qrcode",
    "orderid": "20180202171044007002473",
    "qr_code": "https://qr.alipay.com/fkx03470ljzmqsueedea",
    "paymoney": "0.01",
    "pri_paymoney": "0.01",
    "paystatus": "0",
    "order_time": "1517562644",
    "mch_orderid": "",
    "code_url": "https://qr.alipay.com/fkx03470ljzmqsueedea"
}
```

### 注意事项

- `qr_code` 为二维码链接，需自行生成二维码图片展示给用户；如果是支付宝，可作为跳转支付的链接，如外部APP/H5跳转到支付宝发起支付
- `code_url` 为支付二维码在线地址图片，放到img标签内如：&lt;img src="code_url" /&gt;（code_url是服务器生成的二维码图片，如果自己生成二维码用 qr_code 的内容）
- 商户收银软件需**轮询订单状态**（`pay.comm.query_order`）判断用户是否已完成支付
- 二维码默认 **15分钟** 超时，超时后不可支付
- 测试环境**不发送异步通知**
